Situational Cheat Sheet

Not another command dump.

Most Git cheat sheets start with commands. This one starts with situations. Because when something breaks, you are usually not thinking, “I need a reference for git restore.” You are thinking, “I changed the wrong files. How do I get back safely?”

This project turns common Git panic moments into practical recipes with danger labels, explanations, and safer first steps.

Learning Git? Start with situations.

Git makes more sense when you connect commands to real problems. Each recipe explains when to use a command, what it changes, and what to check before running it.

Using AI to code? Keep Git human-readable.

AI assistants can move fast. Sometimes too fast. This cheat sheet helps you slow down at the right moments: before deleting local files, rewriting history, force-pushing, or accepting a command you do not fully understand.

Git safety rules worth remembering

  1. Inspect before you act.
  2. Prefer reversible commands when working on shared branches.
  3. Use git clean -fdn before git clean -fd.
  4. Use git push --force-with-lease, not plain git push --force.
  5. Use git reflog before assuming work is lost.
  6. Never trust a command just because an AI assistant suggested it.
  7. If secrets were committed, rotate them. Removing them from Git is not enough.

Working with Git in the age of AI assistants

AI can write commands. You still own the repo.

AI coding assistants are excellent at suggesting Git commands, but Git commands are not all equal. Some only inspect your project. Some rewrite history. Some delete files. This cheat sheet is designed for the moment when an assistant says “just run this” and your human brain says, “Wait — what does that do?”
